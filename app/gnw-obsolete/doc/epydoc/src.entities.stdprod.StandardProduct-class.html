Class StandardProduct

The StandardProduct class object's role is to store information on the standard product that are tradable in the market. The StandardProduct class object collaborates with the Market Entity and one or more TradeTranche Entity objects

create_model(self, prefix="")

source code 

The dynamics of the standard product is as follows:

The StandardProduct model assumes that on the one hand it is linked to a Market Entity (as source and sink) and to one or more TradeTranche Entity(s) (each as a sink and a source). Furthermore, the StandardProduct holds an array of position pulp.LpVariable(s) of size len(DISPATCH_PERIODS). Given that the

  • sum(src.out_flow[t], src in srcList) == in_flow[t], and
  • sum(snk.in_flow[t], snk in snkList) == out_flow[t], for all t in DISPATCH_PERIODS

(see method Entity.create_model) the following additional constraints must hold:

  • Inflow/outflow restricted to standard product's delivery period:
    • Restricted inflow volumes: in_flow[t] [MWh] == 0, for t not in DELIVERY_PERIOD
    • Restricted outflow volumes: out_flow[t] [MWh] == 0, for t not in DELIVERY_PERIOD
  • Transacted (sold: position[t] < 0; purchased: position[t] > 0) gas position in [MW]:
    • Inflow/outflow balance equations: position[t]*DISPATCH_PERIODS[t] [MW*h] == CURRENT_POSITION*DISPATCH_PERIODS[t] [MW*h] + out_flow[t] [MWh] - in_flow[t] [MWh], for all t in DISPATCH_PERIODS, i.e., if we sell then in_flow[t] > out_flow[t] and hence, position[t] < 0, if we buy then out_flow[t] > in_flow[t] and position[t] > 0, position[t] == 0, otherwise.
    • Delivery period volume equations: position[t] [MWh] == position[u] [MWh], for all t, u in DELIVERY_PERIOD
  • Sales (position < 0) gas position in [MW]:
    • Non-negativity constraints: shrt_position[t] [MW] >= 0, for all t in DISPATCH_PERIODS (see variable construction in method create_lp_vars).
    • Sales position constraints: shrt_position[t] [MW] >= -position[t] [MW], for all t in DISPATCH_PERIODS
    • Minimum trade size constraint:
      • sum(shrt_position[t]*DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] >= shrt_semi_cont_trigger*MIN_TRADE_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h],
      • sum(shrt_position[t]*DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] <= shrt_semi_cont_trigger*MAX_TRADE_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h],

      where shrt_semi_cont_trigger is a binary variable (i.e., is pulp.LpInteger, 0 <= shrt_semi_cont_trigger <= 1) (--> semi-continuous)

  • Purchase (position > 0) gas position in [MW]:
    • Non-negativity constraints: long_position[t] [MW] >= 0, for all t in DISPATCH_PERIODS (see variable construction in method create_lp_vars).
    • Purchase position constraints: long_position[t] [MW] >= position[t] [MW], for all t in DISPATCH_PERIODS
    • Minimum trade size constraint:
      • sum(long_position[t]*DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] >= long_semi_cont_trigger*MIN_TRADE_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h],
      • sum(long_position[t]*DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] <= long_semi_cont_trigger*MAX_TRADE_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h],

      where long_semi_cont_trigger is a binary variable (i.e., is pulp.LpInteger, 0 <= shrt_semi_cont_trigger <= 1) (--> semi-continuous)

  • Exclusivity constraint: shrt_semi_cont_trigger + long_semi_cont_trigger <= 1, (only one of the variables can be one, redundant constraint)
  • Clip size constraints (if CLIP_SIZE is not None):
    • sum(shrt_position[t]*DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] == shrt_semi_cont_trigger*MIN_TRADE_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] + shrt_clip_count*CLIP_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h]
    • sum(long_position[t]*DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] == long_semi_cont_trigger*MIN_TRADE_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h] + long_clip_count*CLIP_SIZE*sum(DISPATCH_PERIODS[t], t in DELIVERY_PERIOD) [MW*h]

Instance Variable Details [hide private]

CLIP_SIZE

This is a positive floating point number or None representing the standard product clip size in [MW] that can be executed in the market for quantities above the MIN_TRADE_SIZE. If it is set to a positive number any quantity traded above the MIN_TRADE_SIZE will have to be an integral multiple of the CLIP_SIZE. If it is set to None it will be ignored and any quantity above the MIN_TRADE_SIZE can be traded. The equivalent [MWh] clip size is calculated as
  • t_s = DELIVERY_PERIOD[0]
  • t_e = DELIVERY_PERIOD[1]
  • clipSizeMWh [MWh] = CLIP_SIZE*sum(DISPATCH_PERIODS[t]; t in t_s..t_e) [MW*h]

CURRENT_POSITION

Is a floating point number representing the current hedge position of that standard product at the time of optimisation in [MW]. The equivalent [MWh] position is calculated as
  • t_s = DELIVERY_PERIOD[0]
  • t_e = DELIVERY_PERIOD[1]
  • positionMWh [MWh] = CURRENT_POSITION*sum(DISPATCH_PERIODS[t]; t in t_s..t_e) [MW*h]
  • CURRENT_POSITION > 0 for long position
  • CURRENT_POSITION < 0 for short position.
Type:
number

DELIVERY_PERIOD

This is a tuple (t_s, t_e) of index values into the DISPATCH_PERIODS array and define as such the delivery period of the given standard product. It is the user's responsibility to assure the consistency of DISPATCH_PERIODS array contents and the DELIVERY_PERIOD tuple values. This means that the standard product's delivery period must be representable by one or more consecutive dispatch periods given in DISPATCH_PERIODS.
Type:
(number,number)

MAX_TRADE_SIZE

Hypothetical upper limit trade size in [MW] in order to model a semi-continuous variables, such that for a binary variable b and a continuous variable s the following constraint hold:
  • b*MIN_TRADE_SIZE <= s <= b*MAX_TRADE_SIZE

and therefore

  • s == 0, if b == 0, or
  • MIN_TRADE_SIZE <= s <= MAX_TRADE_SIZE, if b == 1.

Note that MAX_TRADE_SIZE should not represent a real upper boundary to s but is used to force s <= 0, if b == 0.

Type:
number (very large)

MIN_TRADE_SIZE

This is a non-negative floating point number representing the minimal trade size in [MW] that can be executed in the market. The equivalent [MWh] minimal trade size is calculated as
  • t_s = DELIVERY_PERIOD[0]
  • t_e = DELIVERY_PERIOD[1]
  • minTradeSizeMWh [MWh] = MIN_TRADE_SIZE*sum(DISPATCH_PERIODS[t]; t in t_s..t_e) [MW*h].
Type:
number
